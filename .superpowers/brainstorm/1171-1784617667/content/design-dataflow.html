三种工具的数据流

同一套核心逻辑，三种不同的数据路径

🔁 通用请求流程（所有功能共用）

用户输入 → SSE 流式发到 /chat → Agent 判断意图 → 调用对应工具 → DeepSeek 生成 → 流式返回前端

📄 工具一：文档总结

① 前端
用户点击上传按钮
→ 选文件 → FormData 发到 /upload
→ 后端返回文件名 + 预览片段
→ 用户确认后发聊天消息："总结这份文档"
② 解析
doc_parser 根据扩展名选择策略：
.txt → 直接读取
.docx → python-docx
.md → 直接读取
.pdf → pdfplumber
统一输出：纯文本字符串
③ AI 总结
文本 + System Prompt → DeepSeek
Prompt 模板："请总结以下文档，输出：
1. 一句话摘要
2. 3-5 个关键要点
3. 关键数据/日期（如有）"

📝 工具二：文案生成

① 选择模板
前端展示模板卡片：
• 📊 周报模板
• 📋 会议纪要模板
• 📢 工作通知模板
用户选择 + 输入关键信息
② 渲染
Jinja2 模板填入用户信息：
"本周完成：{{ tasks }}..."
→ 生成结构化 Prompt
→ 发送给 DeepSeek 润色扩写
③ 输出
格式化 Markdown 文档
前端渲染 + 提供复制按钮
可下载为 .md 文件

🔍 工具三：联网检索

① 触发
Agent 识别到实时信息需求
→ 调用 web_search 工具
→ 提取搜索关键词
② 搜索
DuckDuckGo / Bing Search API
→ 获取 Top 5 结果
→ 提取标题 + 摘要 + URL
③ 整合
搜索结果 + 用户问题 → DeepSeek
→ AI 整合为自然语言回答
→ 末尾附信息来源链接

🧠 上下文记忆机制

正常对话 保留最近 10 轮完整对话 + 前面对话的摘要
触发压缩 当总轮数 > 10 → 取最早的 5 轮 → 发给 DeepSeek 压缩为 1-2 句摘要
压缩结果 摘要存入 memory，旧消息归档。每次请求附带"历史摘要 + 最近 10 轮"
清空记忆 用户点击"新对话"按钮 → POST /clear-memory → 重置所有状态

🛡️ 异常处理设计

📎 文件太大
> 5MB → 前端拦截 + 提示
📄 不支持的格式
→ 返回友好提示 + 支持的格式列表
☁️ API 调用失败
→ 重试 1 次 → 仍失败则提示用户稍后重试
⏱️ 响应超时
→ 30 秒超时 → "回答生成中，请稍候..."
🔑 配置缺失
→ 启动时检查 API Key，缺失则提示配置
📏 输出过长
→ max_tokens 限制 + 前端截断提示
