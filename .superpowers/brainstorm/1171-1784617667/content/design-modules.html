项目文件结构

一眼看清每个文件是干什么的

office-ai-assistant/
├── app.py ← 入口：启动服务器就这一条命令
├── config.py ← 配置：API Key、模型名、模板列表
├── requirements.txt ← 依赖：pip install -r 即可
│
├── agent/ ← 🧠 Agent 大脑
│ ├── core.py ← 对话管理 + 工具调度
│ └── memory.py ← 上下文记忆（滑动窗口 + 摘要）
│
├── tools/ ← 🔧 三件工具
│ ├── doc_parser.py ← 📄 文档解析：txt/docx/md/pdf → 纯文本
│ ├── doc_generator.py ← 📝 文案生成：周报/纪要/通知模板
│ └── web_search.py ← 🔍 联网检索：搜 → 返回 → 整合
│
├── api/ ← 🌐 API 路由
│ └── routes.py ← /chat /upload /summarize /generate
│
└── static/ ← 🎨 前端（全在这）
├── index.html ← 页面结构
├── style.css ← 样式（暗色主题）
└── app.js ← 交互逻辑（SSE + 文件上传）
🧠 Agent 核心
负责什么：
• 接收用户消息，判断意图
• 选择并调用合适工具
• 管理对话历史 + 摘要压缩
• 调用 DeepSeek API（Function Calling）
🔧 工具模块
三个独立工具，各自处理一类任务：
• 文档解析 → 统一输出纯文本
• 文案生成 → Jinja2 模板渲染
• 联网检索 → 搜索 + 结果清洗
🌐 API 路由层
5 个端点：
GET / 前端页面
POST /chat 对话（SSE流式）
POST /upload 文件上传
GET /templates 模板列表
POST /clear-memory 清除记忆
🎨 前端
单页聊天应用，包含：
• 聊天消息列表（支持 Markdown）
• 侧边栏：功能切换 + 文件上传
• 流式输出动画（打字机效果）
• 暗色主题，响应式布局
⚙️ 配置
config.py 统一管理：
• DEEPSEEK_API_KEY
• DEEPSEEK_MODEL（deepseek-chat）
• MAX_OUTPUT_LENGTH
• MEMORY_MAX_ROUNDS
• TEMPLATES 模板注册表
📦 技术依赖
fastapi + uvicorn Web框架
openai DeepSeek SDK（兼容）
python-docx + pdfplumber 文档
jinja2 模板引擎
httpx 联网请求
markdown 渲染
