方案 C：React/Vue + FastAPI 是怎样工作的？

同一个场景——上传会议记录求总结，在方案 C 下的真实流程

⚠️ 开发前你需要做的事（方案 A 不需要）
🔧 环境准备（一次性）
① 安装 Node.js（约 100MB）
② npm create vite@latest → 脚手架项目（几十个文件）
③ npm install → 下载 node_modules（几百 MB）
④ pip install fastapi uvicorn → Python 后端
🖥️ 每次启动需要开两个终端
终端 1 npm run dev → 前端 :5173
终端 2 python app.py → 后端 :8000

还要配 CORS：允许 :5173 跨域访问 :8000
用户上传文件时的请求路线：
你
📎 在 :5173 页面上传文件
1
⚛️ React 组件
状态管理 → 调用 fetch → CORS 预检 → 发到 :8000
2
⚡ FastAPI (:8000)
处理请求 → 调用 DeepSeek → 返回 JSON
3
⚛️ React 再处理
接收 JSON → 更新 state → 重新渲染组件 → 显示结果

方案 A vs 方案 C — 开发体验对比

方案 A 方案 C
启动命令 1 条：python app.py 2 个终端，各 1 条
需安装的环境 仅 Python Python + Node.js
项目文件数（创建时） ~5 个 ~50+ 个（脚手架生成）
前端调试难度 低：刷新页面即可 中：需理解组件状态流
打包部署 复制文件夹即可 npm build → 部署 dist/
适合场景 中小项目 / 快速验证 大型项目 / 团队协作
结论：这个项目，"杀鸡焉用牛刀"
方案 C 是好的技术，但对于一个"轻量"定位的 3 功能助手来说，它带来的复杂度远超收益
