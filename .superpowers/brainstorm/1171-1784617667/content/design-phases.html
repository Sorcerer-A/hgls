P0 · 项目骨架 + 基础对话

目标：python app.py 能跑起来，能聊天
✅ 创建项目结构（app.py + config.py + 各模块目录）
✅ FastAPI 启动 + 静态文件挂载 + index.html 骨架
✅ POST /chat 端点：对接 DeepSeek API，SSE 流式返回
✅ 前端：聊天气泡 + 输入框 + 流式显示 + 发送按钮
✅ 基础异常处理：API Key 检查、超时提示
🎯 里程碑：打开浏览器 → 输入 "你好" → AI 逐字回复

P1 · 核心工具实现

目标：三个工具全部可用
✅ 文档解析器：支持 .txt .docx .md .pdf → 统一纯文本输出
✅ 文件上传：POST /upload 端点 + 前端拖拽上传区域
✅ 文案生成器：3 套 Jinja2 模板 + GET /templates + 模板选择卡片 UI
✅ 联网检索：DuckDuckGo + Serper 双后端 + 策略切换配置
✅ Function Calling：Agent 核心的工具定义 + 工具调度逻辑
🎯 里程碑：上传一份会议记录 → AI 自动识别意图 → 调用文档总结工具 → 输出结构化摘要

P2 · 记忆系统 + 基础优化

目标：E1-4 全部到位
✅ 上下文记忆：10 轮滑动窗口 + 摘要压缩 + /clear-memory 端点
✅ 输出长度限制：max_tokens 配置 + 前端截断提示
✅ 异常捕获完善：6 种场景分类处理 + 友好提示文案
✅ 新对话按钮：清除记忆，重置会话
🎯 里程碑：连续 15 轮对话不丢失上下文 + 任意错误都有中文友好提示

P3 · 界面打磨 + 收尾

目标：交付级品质
✅ 独立视觉设计：调用 frontend-design skill，去大众化配色
✅ 响应式适配：移动端侧栏折叠、触控友好间距
✅ Markdown 渲染：代码高亮、表格、复制按钮
✅ 加载动效：打字机光标、思考中动画
✅ 文案下载：生成的周报/纪要可复制或下载为 .md
🎯 里程碑：完整可用的企业办公 AI 助手，一条命令启动，界面精致

⏱️ 预估工期（单人开发）

阶段 内容 预估
P0 项目骨架 + 基础对话 ~30 min
P1 三个工具 + Function Calling ~45 min
P2 记忆系统 + 优化 + 异常处理 ~30 min
P3 界面打磨 + 响应式 + 动效 ~30 min
合计 ~2-2.5 小时
