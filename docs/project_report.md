# 轻量企业办公 AI 助手 — 项目汇报

> 项目代号：OfficeAI · 开发周期：2026年7月 · 技术栈：Python FastAPI + DeepSeek V4 + SQLite + Vanilla JS

---

## 一、项目概述

基于 DeepSeek V4 大模型 API 开发的轻量级企业办公 AI 助手，提供**文档总结**、**文案生成**、**联网检索**三大核心能力，配有简约 Web 聊天界面。支持动态切换大模型、5 套界面主题、自定义文案模板。

### 核心指标

| 指标 | 数据 |
|------|------|
| 代码提交 | 27 次 |
| 源文件 | 16 个 |
| 后端架构 | FastAPI + SSE 流式输出 |
| 大模型 | DeepSeek V4 + 动态切换 |
| 数据持久化 | SQLite（WAL 模式） |
| 前端 | 原生 HTML/CSS/JS，零构建步骤 |

## 二、核心功能

### 2.1 文档总结

- 支持 **txt / docx / md / pdf** 四种格式
- 自动编码检测（UTF-8 → GBK → GB2312）
- 安全限制：5MB 上限、100 页 PDF 上限、30 秒解析超时
- 摘要 + 关键要点 + 自定义提问

### 2.2 文案生成

- 三套内置模板：**周报**、**会议纪要**、**工作通知**
- 模板卡片化选择，点击即用
- **用户可自定义模板**（Jinja2 语法，设置面板编辑）
- AI 润色 + Markdown 格式化输出

### 2.3 联网检索

- Serper API（Google 搜索），结果质量远超免费方案
- RAG 模式：搜索 → 注入 Prompt → DeepSeek 整合
- 策略模式双后端（Serper + Bing），自动故障切换
- 结果缓存（TTL 1 小时）+ 调用冷却

### 2.4 上下文记忆

- 滑动窗口（10 轮完整对话）+ 摘要压缩
- SQLite 持久化，重启不丢失
- 异步后台压缩，不阻塞用户请求
- 7 天自动清理

## 三、技术架构

```
浏览器 (HTML/CSS/JS)
    │  SSE 流式 / Fetch
    ▼
FastAPI 路由层 (/chat /upload /templates /settings /clear-memory)
    │
    ├── Agent 核心 (Function Calling + 工具调度 + 流式输出)
    │     ├── 文档解析器 (python-docx / pdfplumber)
    │     ├── 文案生成器 (Jinja2 + 自定义模板)
    │     └── 联网检索 (Serper / Bing 策略模式)
    │
    ├── 记忆系统 (SQLite + WAL + asyncio.Lock)
    │
    └── DeepSeek V4 API (OpenAI 兼容 SDK)
```

## 五、工程实践

### 5.1 错误处理

- 指数退避重试（最多 3 次）
- 不可重试错误分类（401/400）vs 可重试（429/503/超时）
- 降级策略：搜索后端自动切换、模型不可用友好提示
- 文档解析异常分类处理（6 种场景）

### 5.2 安全措施

- API Key 环境变量管理（`.env` + `python-dotenv`）
- DOMPurify XSS 防护（Markdown 渲染白名单）
- PDF 炸弹防护（页数上限 + 解析超时）
- SQLite WAL 模式保证并发安全

### 5.3 配置灵活性

- 动态大模型切换（Base URL / API Key / Model）
- 用户可自定义三套文案模板
- 设置面板即时生效

## 八、技术亮点

1. **搜索方案演进**：DuckDuckGo → Bing HTML 抓取 → Serper (Google) → 最终采用 RAG 直接注入，解决 Function Calling XML 泄露问题
2. **Python 3.8 兼容**：使用 `from __future__ import annotations` 和 `loop.run_in_executor` 适配旧版 Python
3. **零构建步骤**：原生 HTML/CSS/JS，`python app.py` 一键启动
4. **策略模式**：搜索后端可插拔，配置即可切换
