## 二、界面设计

### 2.1 布局架构

左右分栏式设计，左侧功能面板 + 右侧聊天区：

```
┌──────────┬────────────────────────────┐
│ ⚙️ 设置   │                            │
│ + 新对话  │     📝 AI 回复消息区        │
│ 🗑️ 清除   │     (Markdown 渲染)        │
│          │                            │
│ 💬 自由   │     用户消息气泡            │
│ 📄 总结   │     AI 回复气泡 + 操作按钮  │
│ 📝 文案   │                            │
│ 🔍 检索   │                            │
│          ├────────────────────────────┤
│ 📎 上传   │  [输入框.........] [发送]   │
└──────────┴────────────────────────────┘
```

### 2.2 5 套主题系统

| 主题 | 配色 | 风格描述 |

1. **跳动圆点** — AI 消息区显示 3 个弹跳动画点
2. **进度条** — 顶部 2px 琥珀色流动进度条
3. **状态文字** — `思考中...` → `⚡ 正在联网搜索...` → `正在输出...`

### 3.2 文案生成

**操作**：点击「📝 文案生成」→ 选择模板卡片 → 输入内容

问题风险：第三方支付SDK版本过旧需升级，前端人手不足

### 3.3 联网检索

**操作**：点击「🔍 联网检索」→ 输入查询

```
Q: 最近一周AI领域重要新闻
A: 基于搜索结果（截至2026年7月22日），本周热点包括：
   1. The Guardian (7/20)：中国AI芯片进展引发硅谷关注
   2. 习近平发起新AI联盟
   3. ...
```

**技术实现**：
- Serper API（Google 搜索）+ `tbs=qdr:w` 近一周过滤
- 自动注入当前日期到搜索查询和系统提示
- RAG 模式：搜索 → 注入 Prompt → DeepSeek 整合

### 3.4 上下文记忆

- **滑动窗口**：保留最近 10 轮完整对话
- **摘要压缩**：超过 10 轮时异步后台压缩前 5 轮
- **SQLite 持久化**：WAL 模式，重启不丢失
- **7 天自动清理**：过期会话自动删除

### 3.5 设置面板

**⚙️ 入口**：左侧顶部齿轮图标

| 功能 | 说明 |
|------|------|
| API 配置 | Base URL / API Key / Model 可动态修改 |
| 主题切换 | 5 套主题即时预览 + 一键切换 |
| 模板编辑 | 周报/纪要/通知的 Jinja2 模板可自定义 |

## 四、技术架构

```
浏览器 (HTML/CSS/JS + SSE)
    │
    ▼
FastAPI 路由层
    │
    ├── Agent 核心 ── 对话管理 + Function Calling + 流式输出 + 重试
    │     ├── 文档解析器 (python-docx / pdfplumber)
    │     ├── 文案生成器 (Jinja2 + 自定义模板)
    │     └── 联网检索 (Serper / Bing 策略模式)
    │
    ├── 记忆系统 (SQLite + WAL + asyncio.Lock)
    │
    └── DeepSeek V4 API (OpenAI 兼容 SDK · 动态模型切换)
```
